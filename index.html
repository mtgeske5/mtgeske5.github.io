Algorithms & Data Structures — Reinforcement Learning (DQN)

Python Deep Q-Learning Prioritized Experience Replay Tradeoffs
View Project Files (GitHub)

Narrative

I decided to choose an artifact that I originally spent a lot of time working on because of the complexity involved. I chose the learning based Treasure Hunt game that I created in the previous fall in CS-370. This program involved an understanding of a Deep Q learning network, and how the agent is able to learn over time based on the strategies provided. The implementation of a DQN allowed for an AI agent to progress through a maze and eventually learn all the ins and outs in order to complete the maze with 100% accuracy no matter where the agent started the maze. The original version relied on standard experience using uniform random sampling as the method to train the AI agent’s neural network.

This artifact demonstrates my ability to design and evaluate the computing solutions using algorithmic principles and data structures. The project involves a lot of strategy as to discovering which is the most efficient method of neural understanding between uncertainties, reward based, and iterative optimization.

To enhance this project, I replaced the uniform experience replay with prioritized experience replay (PER). Originally, the training samples were drawn from memory and each past experience was treated equally. The PER method assigns higher sampling probability to experiences with a larger error. This completely changes the way the agent thinks and understands previous decisions made. It requires a new data structure to track the agent’s previous decisions and prioritize them with a weighted sample.

Both methods of learning may lead to the same result in the end, but algorithmically this enhancement improves the learning efficiency because the agent is able to replay more informative experiences even more frequently. The downside may be the agent’s ability to maintain diversity in it’s samples, but it introduces dynamic priority updates based on prediction error. The program originally allows me to change the parameters to adjust the agent’s neural network, but by changing the sampling strategy I am able to demonstrate how data structures and algorithmic trade offs can more efficiently and accurately complete the task.

The overall goal of this enhancement was to showcase my progress in designing and evaluating computing solutions with the use of algorithmic principles and appropriate data structures. The prioritized replay buffer addition required integrating a different strategy of sampling that balances complexity and performance.

The enhancement required me to look at the program from a different perspective by changing the data structure instead of the parameters. By implementing this method, I was able to further my understanding of how to directly influence the behavior and system performance. One of the main challenges was understanding the program as a whole because originally, I was only able to edit a specific part of the program that did not involve adjusting the data structure. I didn’t want to rewrite the program that adjusted the goal, but integrating the prioritize replay was an option to keep the program’s intentions while upgrading it’s efficiency. This process increased my knowledge in algorithmic improvements and it increased my understanding of technological trade-offs. Balancing these factors are real-life challenges that I will face and this enhancement highlights my ability to logically understand the importance of algorithmic performance and system behavior making this a strong addition to my ePortfolio.

Databases — CRUD Enhancement with REST API (Flask)

Python MongoDB CRUD Flask REST API Validation
View Project Files (GitHub)

Narrative

The artifact I chose was from the CS-340 class where I worked on creating a database with the CRUD strategy. The original artifact includes an AnimalShelter class that used the basic create, read, update, and delete operations. I used basic Python and MongoDB skills to complete the required database actions. To enhance this database, I expanded it to include a REST style API layer with Flask. It supports externally accessible database operations through HTTP endpoints.

I thought this would be a great artifact to include in my ePortfolio because of the overall practicality of databases and how I was able to increase its capabilities. Databases are used in just about every software system, so being able to design and improve them is a practice that should be mastered for future employment opportunities. My enhancements for this specific artifact include isolating the database access logic within the AnimalShelter class by introducing application layers and input validation. Databases are more than likely not accessed through direct application code, so being able to access through APIs is definitely more industry standard. I added the REST interface to show the CRUD operations, as well as, including input validation and centralized error handling.

In module 1, I proposed to make these changes because I think a database should be more robust with more accesses and the ability to be reused. I believe I improved the abilities and efficiencies from the original artifact. I was able to learn a lot on databases as a whole by thinking backwards and trying to research how a database should look professionally and adding those properties to the enhanced artifact.

From my understanding, separating database logic and application logic improves efficiency and usage. The REST API layer really clarified how databases are commonly accessed in larger real-world environments. I think the biggest challenge was the error handling and figuring out the best way for the database to handle invalid input or database connectivity issues. All of this can be squared away by designing with security in mind because poor error handling could lead to vulnerabilities.

Addressing these challenges helped strengthen my understanding of defensive programming and the security implications of exposing database operations through an application interface.
